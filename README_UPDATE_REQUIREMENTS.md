# README 更新要求

## 目标

保证 README 与仓库实际内容一致，便于课程学习、实验复现和后续维护。

## 适用范围

当出现以下任一情况时，必须更新 README：

1. 新增、删除或重命名实验目录。
2. 新增、删除或重命名关键数据文件、脚本文件或 Notebook。
3. 依赖环境或运行方式发生变化。
4. 实验内容、说明文字或学术诚信要求发生变化。

## 必更内容清单

每次更新 README，至少完成以下检查：

1. 项目简介与当前课程/仓库用途一致。
2. 目录结构包含最新的 Lab 文件夹与关键文件。
3. 环境准备中的 Python 版本与依赖安装命令可用。
4. 实验内容概览覆盖所有已有实验模块。
5. 使用方式中的脚本和 Notebook 启动命令可执行。
6. 学术诚信与许可证说明保持存在且表述清晰。
7. README 尾部“最后更新”日期同步为本次修改日期。

## 编写规范

1. 使用简洁、可执行的说明，避免空泛描述。
2. 结构建议固定为：简介、目录结构、环境准备、实验概览、使用方式、维护说明、学术诚信、许可证。
3. 命令示例应直接可复制运行。
4. 如仓库存在中英文双版本内容，README 应明确说明。
5. 不在 README 中记录过于细碎的过程日志，过程记录建议放到独立文档。

## 验收标准

README 更新完成后，需满足：

1. 目录、文件名与仓库实际一致。
2. 关键命令在当前环境下无明显错误。
3. 不包含失效路径、过时文件名或错误实验编号。
4. 文档无明显错别字，标题层级清晰。

## 建议流程

1. 先确认仓库结构变化。
2. 再修改 README 对应章节。
3. 最后执行一次自查（按本文件“必更内容清单”和“验收标准”逐项核对）。
